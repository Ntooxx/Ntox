# Programming — Open Source & Collaboration

## Purpose

The practice of building software in the open. Open source is not just free code — it is a philosophy of collaboration, transparency, and shared ownership. Collaboration is the force multiplier.

## Core Principle

> Open source is not just free code.
> It is a philosophy of collaboration.
> Transparency.
> Shared ownership.
> Meritocracy of ideas.
> The best idea wins, regardless of who proposes it.

## Open Source Models

| Model | Description |
|-------|-------------|
| **Benevolent Dictator** | One person makes final decisions |
| **Meritocracy** | Contributors earn commit rights |
| **Foundation** | Governed by foundation |
| **Corporate** | Company controls project |
| **Community** | Distributed decision making |

## Contributing to Open Source

### Finding Projects
- Scratch your own itch
- Find projects you use
- Look for "good first issue"
- Join communities

### Making Contributions
1. Read contributing guidelines
2. Fork the repository
3. Create a branch
4. Make changes
5. Write tests
6. Update documentation
7. Submit a pull request

### Code Review Etiquette
- Be respectful
- Be constructive
- Explain your reasoning
- Accept feedback gracefully
- Learn from discussions

## Maintaining Open Source

### Project Management
- Clear README
- Contributing guidelines
- Code of conduct
- Issue templates
- Pull request templates
- Release notes

### Community Building
- Respond to issues promptly
- Welcome new contributors
- Recognize contributions
- Communicate clearly
- Be inclusive

### Sustainability
- Funding (sponsors, grants)
- Governance
- Succession planning
- Burnout prevention
- Work-life balance

## Git Best Practices

### Commit Messages
- Subject line: imperative mood, <50 chars
- Body: explain what and why, not how
- Reference issues

### Branching
- Feature branches
- Pull requests
- Code review
- Merge or rebase

### Collaboration
- Fork and pull
- Shared branches
- Protected branches
- CI/CD integration

## Common Collaboration Errors

| Error | Description | Prevention |
|-------|-------------|------------|
| **No documentation** | Nobody can contribute | Write docs |
| **Hostile environment** | People leave | Be welcoming |
| **Burnout** | Maintainers burn out | Share responsibility |
| **No governance** | Decisions are unclear | Define process |
| **No contributing guide** | People don't know how to help | Write guide |
| **Ignoring issues** | Community feels ignored | Respond promptly |

## Integration

- **Code Review**: Open source relies on review
- **Documentation**: Documentation enables contribution
- **Testing**: Tests ensure quality
- **Architecture**: Architecture affects contribution
- **Community**: Community is the heart of open source
- **License**: License defines usage
