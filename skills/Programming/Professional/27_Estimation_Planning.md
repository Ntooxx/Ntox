# Programming — Estimation & Planning

## Purpose

The art of predicting how long things will take. Not just guessing — structured prediction based on experience, decomposition, and honest uncertainty. Estimation is hard, but essential.

## Core Principle

> Estimation is not prediction.
> Estimation is understanding.
> The act of estimating forces you to think.
> The estimate itself is secondary.
> The thinking is the value.

## Estimation Framework

### Step 1: Decompose
- Break task into smaller pieces
- Each piece should be 1-2 days max
- Identify dependencies
- Identify unknowns

### Step 2: Estimate Each Piece
- Best case: everything goes right
- Likely case: normal obstacles
- Worst case: things go wrong
- Use historical data if available

### Step 3: Add Up
- Sum best case estimates
- Sum likely case estimates
- Sum worst case estimates
- Apply uncertainty multiplier

### Step 4: Communicate
- Give ranges, not point estimates
- State assumptions clearly
- Identify risks and unknowns
- Plan for re-estimation

## Estimation Techniques

### 1. Analogous Estimation
- Use similar past projects
- Adjust for differences
- Quick but imprecise
- Good for high-level estimates

### 2. Parametric Estimation
- Use mathematical models
- Lines of code, function points
- More precise but requires data
- Good for large projects

### 3. Three-Point Estimation
- Best case (B)
- Likely case (L)
- Worst case (W)
- Expected = (B + 4L + W) / 6

### 4. T-Shirt Sizing
- XS, S, M, L, XL, XXL
- Relative, not absolute
- Good for backlog grooming
- Quick and collaborative

### 5. Planning Poker
- Team estimates together
- Discuss differences
- Consensus-based
- Good for team alignment

## Common Estimation Errors

| Error | Description | Prevention |
|-------|-------------|------------|
| **Anchoring** | First estimate influences later | Independent estimation |
| **Optimism bias** | Underestimating difficulty | Add buffer |
| **Unknown unknowns** | Not accounting for surprises | Add uncertainty |
| **Pressure** | Stakeholders pressure for lower estimates | Be honest |
| **Not re-estimating** | Sticking with original estimate | Re-estimate regularly |
| **Ignoring technical debt** | Not accounting for existing debt | Factor in debt |
