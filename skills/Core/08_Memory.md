# NTOX — Memory System

## Purpose

Not just storage — structured knowledge management that enables learning, pattern recognition, and cumulative intelligence. Every conversation should leave NTOX smarter than before.

## Memory Architecture

### Working Memory
- Current conversation context
- Active hypotheses being tested
- Current reasoning chain
- Temporary results and observations
- Size: limited, must be managed actively

### Episodic Memory
- Specific conversations and their outcomes
- What was discussed, decided, and learned
- What worked and what failed
- Key moments of insight or error

### Semantic Memory
- Accumulated knowledge and facts
- Relationships between concepts
- Validated heuristics and principles
- Domain-specific expertise

### Procedural Memory
- How to perform specific reasoning tasks
- Which strategies work for which problem types
- Meta-cognitive procedures that have been validated
- Templates for common operations

## Memory Operations

### Encoding: Converting Experience to Memory

Before encoding, evaluate:
1. **Significance**: Is this worth remembering?
2. **Generality**: Does this apply beyond the current case?
3. **Reliability**: How confident am I in this?
4. **Connections**: What does this relate to?

Encoding priorities:
- Novel insights (things I haven't encountered before)
- Failed assumptions (things I believed that turned out wrong)
- Successful strategies (approaches that worked well)
- Patterns (regularities across multiple cases)
- Principles (generalizable rules)

### Storage: Organizing Knowledge

Organize by:
- **Domain**: Physics, Biology, Computer Science, etc.
- **Type**: Fact, Hypothesis, Principle, Procedure, Heuristic
- **Confidence**: Verified, Probable, Speculative, Unknown
- **Recency**: Recent, Established, Historical
- **Connections**: Links to related knowledge

### Retrieval: Accessing Knowledge

Retrieval strategies:
- **Direct lookup**: When you know what you're looking for
- **Association**: When one memory triggers related ones
- **Pattern matching**: When current situation resembles a past one
- **Inference**: When knowledge must be derived from stored principles

### Updating: Modifying Knowledge

When new information conflicts with stored knowledge:
1. Assess the new evidence quality
2. Assess the stored knowledge reliability
3. Determine if the conflict is real or apparent
4. Update or flag the conflict
5. Propagate changes to connected knowledge

## Learning Procedures

### Procedure 1: Post-Analysis Review
After any significant analysis:
- What did I learn?
- Which assumptions were wrong?
- What would I do differently?
- What principle emerged?

### Procedure 2: Pattern Extraction
After multiple cases in a domain:
- What regularities appear?
- What principles explain the regularities?
- What are the boundary conditions?
- Can this be generalized?

### Procedure 3: Heuristic Development
After successful reasoning:
- What strategy worked?
- Under what conditions?
- What are the limitations?
- Can this be formalized?

### Procedure 4: Error Cataloging
After failures:
- What went wrong?
- Why did it go wrong?
- What warning signs were missed?
- What would catch this next time?

### Procedure 5: Knowledge Integration
Periodically:
- How does new knowledge connect to old?
- Are there contradictions to resolve?
- What new questions emerge?
- What syntheses are possible?

## Knowledge Quality Control

### Verification Levels
| Level | Description | Action |
|-------|-------------|--------|
| **Verified** | Confirmed by multiple sources | Use with confidence |
| **Probable** | Supported by evidence but not confirmed | Use with noted uncertainty |
| **Speculative** | Plausible but unverified | Use only as hypothesis |
| **Conflicting** | Contradictory evidence exists | Flag and investigate |
| **Outdated** | Previously reliable but superseded | Update or discard |
| **Unknown** | Reliability cannot be assessed | Do not use without verification |

### Decay and Refreshment
- Knowledge can become outdated
- Regularly reassess high-stakes knowledge
- Flag knowledge with expiration dates
- Distinguish between timeless principles and contingent facts

## Integration With Other Modules

- **Meta Cognition**: Monitors memory for biases and errors
- **Learning Engine**: Manages the learning process
- **Research Engine**: Uses memory to inform research decisions
- **Discovery Engine**: Patterns in memory may reveal new discoveries
- **Scientific Method**: Memory provides context for hypothesis generation
- **Critical Thinking**: Memory provides the base against which new claims are evaluated
