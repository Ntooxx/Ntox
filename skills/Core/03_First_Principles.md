# NTOX — First Principles Thinking

## Triggers
- "first principles"
- "fundamental truth"
- "deconstruct"
- "from scratch"
- "axioms"
- "break it down"

## Purpose

Decompose every problem to its most fundamental truths. Then reason up from there. Never reason by analogy alone — analogies are useful for communication but dangerous for reasoning.

## The Axiom

> Everything that is true is true because of something more fundamental.
> Find that something more fundamental. Then you own the problem.

## Core Procedure

### Step 1: Identify the Current Model
- What is the current explanation for this phenomenon?
- What assumptions does it contain?
- Where did these assumptions come from?
- Who benefits from this framing?

### Step 2: Decompose to Axioms
For each assumption, ask:
- Is this empirically verified?
- Is this a definition or a discovery?
- Is this contingent (could be otherwise) or necessary (must be this way)?
- What evidence would falsify this?

### Step 3: Identify True Primitives
What are the irreducible components?
- Physical: matter, energy, space, time, information
- Mathematical: sets, functions, logic, probability
- Computational: computation, memory, communication
- Economic: scarcity, incentives, exchange, value
- Biological: replication, selection, metabolism, signaling

### Step 4: Rebuild From Primitives
Using only verified primitives:
- What can be constructed?
- What constraints emerge naturally?
- What is impossible?
- What is surprising?

### Step 5: Validate the Rebuilt Model
- Does it explain the original phenomenon?
- Does it make different predictions than the old model?
- Is it simpler? More general? More falsifiable?
- What does it predict that the old model doesn't?

## First Principles by Domain

### Physics
- Conservation laws (energy, momentum, charge)
- Thermodynamics (entropy increases, no perfect conversion)
- Quantum mechanics (uncertainty, superposition, entanglement)
- Relativity (speed of light limit, equivalence of mass-energy)
- Symmetry and invariance

### Mathematics
- Logical consistency
- Completeness limitations (Gödel)
- Computational complexity (P vs NP)
- Information theory (Shannon limits)
- Probability axioms

### Computation
- Church-Turing thesis
- Computational complexity classes
- Halting problem
- No free lunch theorems
- Bounded rationality

### Biology

- Central dogma (DNA → RNA → Protein)

- Data processing inequality

## Common Violations of First Principles

| Violation | Example | First Principle Correction |
|-----------|---------|---------------------------|
| **Argument from authority** | "Expert X says..." | What is the evidence? |
| **Tradition** | "We've always done it this way" | Does this still hold? |
| **Analogy** | "It's like..." | Is the mapping valid? |
| **Complexity bias** | "It must be complicated" | What is the simplest explanation? |
| **Emergence denial** | "It's just X" | Are there emergent properties? |
| **Reductionism excess** | "It's only atoms" | What do higher levels add? |
| **Teleology** | "It exists for a purpose" | What mechanism produced it? |
| **Naturalistic fallacy** | "It's natural, so it's good" | Natural ≠ optimal |

## The First Principles Checklist

For any claim, run through:

1. **Source**: Where did this claim originate?
2. **Evidence**: What empirical support exists?
3. **Assumptions**: What must be true for this claim to hold?
4. **Alternatives**: What other explanations exist?
5. **Falsifiability**: What would disprove this?
6. **Primitives**: Can this be derived from more fundamental truths?
7. **Consistency**: Does this contradict established knowledge?
8. **Parsimony**: Is there a simpler explanation?

## Integration With Other Modules

- **Scientific Method**: First principles provides the foundation; scientific method provides the procedure
- **Critical Thinking**: First principles identifies what to question; critical thinking provides the tools to question it
- **Discovery Engine**: First principles thinking often reveals that "known" truths are actually assumptions
- **Systems Thinking**: First principles at the system level reveals emergent properties that reductionism misses
- **Mathematical Thinking**: Mathematics is the language of first principles — if you can't express it mathematically, you may not understand it well enough

## The First Principles Mantra

> Never accept a conclusion without understanding the chain of reasoning that produced it.
> Never trust a chain of reasoning without verifying each link.
> Never verify a link without knowing what it connects to.
> Start from the bottom. Build up. Test at every level.
