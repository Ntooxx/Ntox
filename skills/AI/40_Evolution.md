## Evolution Framework

### The Darwinian Algorithm
1. **Variation**: Generate diverse candidates
2. **Selection**: Keep the best
3. **Reproduction**: Create next generation
4. **Repeat**: Iterate until satisfied

### Key Concepts
- **Fitness**: How well an organism solves its problem
- **Selection**: Differential survival and reproduction
- **Mutation**: Random changes
- **Crossover**: Combining parents
- **Population**: Set of competing organisms
- **Generation**: One iteration of the algorithm

### Genetic Algorithm
```
Initialize population
While not satisfied:
    Evaluate fitness
    Select parents
    Create offspring (crossover + mutation)
    Replace population
Return best individual
```

## Evolutionary Principles

### 1. Variation is Essential
Without variation, there is nothing to select. Variation can come from:
- Random mutation
- Sexual reproduction (crossover)
- Horizontal gene transfer
- Genetic drift

### 2. Selection Drives Improvement
Selection can be:
- **Natural selection**: Environmental pressures
- **Sexual selection**: Mate choice
- **Artificial selection**: Human choice
- **Quality selection**: Algorithmic evaluation

### 3. Fitness landscapes
- **Peaks**: Optimal solutions
- **Valleys**: Suboptimal solutions
- **Plateaus**: Flat regions with no gradient
- **Ridges**: Narrow paths to peaks

### 4. Local vs Global Optima
- Evolution often gets stuck on local peaks
- High mutation rate can escape local optima
- Sexual reproduction can jump across valleys
- Island models maintain diversity

### 5. Co-evolution
- Species evolve in response to each other
- Arms races create escalating complexity
- Red Queen hypothesis: must keep evolving to stay in place
- Can lead to cycles or stability

## Common Evolution Errors

| Error | Description | Prevention |
|-------|-------------|------------|
| **Premature convergence** | Losing diversity too early | Maintain diversity |
| **Stagnation** | No improvement | Increase selection pressure |
| **Overfitting** | Evolving to noise | Validate on held-out data |
| **Bloat** | Unnecessary complexity | Parsimony pressure |
| **Deception** | Fitness function misleads | Careful fitness design |
